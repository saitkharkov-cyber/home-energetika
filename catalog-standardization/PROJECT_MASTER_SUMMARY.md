# Catalog Standardization

## Назначение подсистемы

**Catalog Standardization** — подсистема Home-Energetika, отвечающая за управление качеством каталога интернет-магазина.

Основная задача подсистемы — создание, поддержание и развитие единого стандарта хранения характеристик товаров независимо от производителя, категории оборудования или источника данных.

Результатом работы является стандартизированный каталог OpenCart, пригодный для использования всеми остальными подсистемами проекта.

---

# Место в архитектуре Home-Energetika

```text
                   Home-Energetika
                           │
        ┌──────────────────┴──────────────────┐
        │                                     │
        ▼                                     ▼
Catalog Standardization               Pump Selector
        │                                     ▲
        └──────── Стандартизированные данные ─┘
```

Подсистема не занимается алгоритмами подбора оборудования.

Её ответственность заканчивается передачей качественных, стандартизированных данных.

---

# Основные задачи

Подсистема отвечает за:

* стандартизацию характеристик товаров;
* унификацию атрибутов;
* устранение дублей;
* разработку правил хранения данных;
* импорт данных производителей;
* автоматизацию обработки каталогов;
* генерацию SQL для массовой миграции данных;
* контроль качества каталога.

---

---

# Архитектура обработки данных

Каждый производитель проходит одинаковый процесс.

```text
Источник данных
        │
        ▼
Парсинг
        ▼
Нормализация
        ▼
Entity Matching
        ▼
Стандартизация атрибутов
        ▼
Проверка
        ▼
Генерация SQL
        ▼
Импорт в OpenCart
```

Архитектура pipeline описывается документом **DATA_PIPELINE.md**.

---

# Entity Matching

Практика показала, что основной сложностью интеграции данных является не качество характеристик производителя, а корректное сопоставление товаров между различными источниками.

Главная задача этого этапа — построение надежного **Canonical Model Key**.

До успешного сопоставления товаров сравнение характеристик считается недостоверным.

# Принципы стандартизации

## Standard First

Сначала утверждается стандарт хранения данных.

После этого выполняются изменения каталога.

---

## Source of Truth

Правила хранения характеристик определяются документом:

```text
CATALOG_STANDARD.md
```

---

## Repeatable Process

Любая массовая стандартизация должна быть полностью воспроизводимой.

Каждый атрибут проходит единый процесс:

1. Анализ.
2. Классификация значений.
3. Выявление исключений.
4. Утверждение стандарта.
5. Проверочный SELECT.
6. UPDATE.
7. Контрольная проверка.
8. Обновление документации.

---

# Развитие подсистемы

Подсистема развивается по двум независимым направлениям:

1. Расширение поддержки новых производителей.
2. Расширение поддержки новых категорий оборудования.

Новые производители и новые категории проходят единый процесс стандартизации без изменения архитектуры подсистемы.

---

# Главный принцип

Качество алгоритмов ограничено качеством входных данных.

Поэтому первичной задачей подсистемы является создание надежного, единообразного и воспроизводимого каталога, который может использоваться всеми сервисами Home-Energetika.

# Pump Selector Contract

## 1. Предусловие

`PumpSelectorCacheBuilder` предполагает, что `Catalog Standardization` уже выполнила нормализацию характеристик товаров.

Нормализация не входит в ответственность Builder и Selector.

---

## 2. Назначение

Настоящий документ определяет контракт данных между:

- Catalog Standardization;
- PumpSelectorCacheBuilder;
- PumpSelector.

Подборщик никогда не анализирует "сырые" значения атрибутов товаров.

Он работает исключительно со стандартизированными значениями, подготовленными подсистемой Catalog Standardization.

Все требования к данным описаны в настоящем контракте.

Контракт считается выполненным, если все обязательные атрибуты присутствуют и соответствуют требованиям настоящего документа.

При соблюдении настоящего контракта Selector гарантирует корректную работу без дополнительной обработки данных.

---

## 3. Общие требования

Catalog Standardization обязана обеспечить наличие стандартизированных атрибутов в карточке товара OpenCart.

Все значения должны быть приведены к единому формату до построения cache подборщика.

Selector не выполняет:

- парсинг строк;
- извлечение единиц измерения;
- исправление форматов;
- преобразование значений;
- угадывание отсутствующих данных.

При нарушении контракта товар исключается из cache.

---

## 4. Контракт атрибутов

Builder использует `attribute_id` как источник истины.

Названия атрибутов приведены исключительно для удобства чтения документации.

| Attribute ID | Атрибут OpenCart                        | Поле кэша          | Тип     | Единица / значения | Обязательный |
|--------------|-----------------------------------------|--------------------|---------|--------------------|--------------|
|      12      | Максимальный напор (м)                  | `max_head_m`       | Decimal | м                  | ✅           |
|      13      | Максимальная производительность (л/мин) | `max_flow_l_min`   | Integer | л/мин              | ✅           |
|      15      | Напряжение питания (В)                  | `voltage`          | Integer | 220 / 380          | ✅           |
|      44      | Диаметр насоса (мм)                     | `pump_diameter_mm` | Decimal | мм                 | ✅           |

### Примечания

- `attribute_id` является архитектурным контрактом.
- Builder использует только `attribute_id`.
- Поиск по названиям атрибутов запрещён.
- Переименование атрибутов OpenCart не должно влиять на работу подборщика.

---

## 5. Необязательные атрибуты

Раздел будет расширяться по мере развития подборщика.

---

## 6. Требования к данным

Все числовые значения должны храниться без единиц измерения.

### Правильно

```text
max_head_m = 90
max_flow_l_min = 30
pump_diameter_mm = 76.2
voltage = 220
```

### Неправильно

```text
90 м
1,8 куб/час
30 л/мин
3"
220 В
```

---

## 7. Назначение cache

Таблица `pump_selector_product` является служебной cache-таблицей подборщика.

Её единственная задача — ускорение поиска и ранжирования подходящих товаров.

---

## 8. Содержимое cache

В cache допускается хранить только данные, необходимые алгоритму подбора.

Например:

- нормализованные характеристики;
- `brand_priority`;
- `is_eligible`;
- `product_price`;
- технические поля отбора.

---

## 9. Данные отображения

Источник истины (Source of Truth) для отображаемых товарных данных — стандартные таблицы OpenCart.

После выбора `product_id` Selector обязан получать данные для отображения из стандартных моделей OpenCart.

В cache не должны использоваться как источник отображения:

- `name`;
- `manufacturer`;
- `image`.

Поле `product_price` является техническим полем cache и используется исключительно для быстрого ранжирования кандидатов.

Отображаемая пользователю цена должна определяться стандартными механизмами OpenCart на основе актуальных данных каталога.

> **Важно**
>
> Единственным Source of Truth для отображаемых товарных данных являются стандартные таблицы OpenCart.

---

## 10. Ответственность Catalog Standardization

Catalog Standardization обязана:

- нормализовать характеристики;
- привести единицы измерения;
- использовать канонические `attribute_id`;
- подготовить каталог для Cache Builder.

---

## 11. Ответственность PumpSelectorCacheBuilder

PumpSelectorCacheBuilder обязан:

- проверить наличие обязательных атрибутов;
- проверить базовые условия отбора;
- построить cache;
- исключить товары, нарушающие контракт;
- сформировать статистику построения cache.

Builder не исправляет данные каталога.

---

## 12. Гарантии контракта

При соблюдении настоящего контракта Selector гарантирует:

- отсутствие дополнительной нормализации;
- отсутствие парсинга строк;
- использование только cache;
- корректную работу алгоритма подбора.

---

## 13. Допустимые изменения

Допускается изменение:

- SQL;
- структуры Builder;
- алгоритма построения cache;
- состава статистики;
- вычисления `brand_priority`;
- внутренней реализации cache.

Не допускается изменение:

- смысла обязательных атрибутов;
- Source of Truth;
- ответственности компонентов;
- архитектурного контракта между Builder и Selector.

---

## 14. Нарушение контракта

Товар не попадает в cache при отсутствии хотя бы одного обязательного атрибута.

Причина исключения должна фиксироваться в статистике Builder.

При нарушении контракта:

- товар исключается из cache;
- Selector не пытается исправить данные;
- отсутствие товара считается корректным поведением системы.

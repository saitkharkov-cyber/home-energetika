# Deployment

## 1. Назначение

Документ описывает порядок установки, обновления и проверки работоспособности Pump Selector.

Документ предназначен для разработчиков и администраторов проекта.

---

## 2. Предварительные требования

Перед установкой должны быть выполнены следующие условия:

- OpenCart установлен и работает;
- каталог товаров заполнен;
- Catalog Standardization выполнена;
- обязательные атрибуты присутствуют;
- структура БД соответствует проекту.

---

## 3. Установка

### 3.1. Код

Развернуть файлы модуля в каталог проекта.

### 3.2. База данных

Выполнить SQL-скрипт установки:

```text
pump-selector/sql/install.sql
```

Он создаёт служебную таблицу:

```text
pump_selector_product
```

Если таблица уже существует, структура должна соответствовать текущей версии проекта.

---

## 4. Построение cache

После установки необходимо выполнить полную перестройку cache.

Запускается:

```text
PumpSelectorCacheBuilder
```

Результатом должна стать заполненная таблица:

```text
pump_selector_product
```

Builder возвращает статистику вида:

```php
array(
    'total_scanned' => ...,
    'eligible_inserted' => ...
)
```

---

## 5. Проверка установки

После успешной сборки cache необходимо проверить:

- таблица `pump_selector_product` существует;
- таблица содержит записи;
- `eligible_inserted > 0`;
- обязательные поля заполнены;
- `brand_priority` рассчитан;
- отсутствуют ошибки PHP.

---

## 6. Проверка подборщика

После установки рекомендуется выполнить несколько тестовых подборов.

Необходимо убедиться, что:

- рекомендации формируются;
- карточки товаров открываются;
- отображается актуальная цена;
- отображается актуальное наличие;
- отображается правильный производитель.

---

## 7. Обновление

При изменении:

- структуры cache;
- обязательных атрибутов;
- алгоритма Builder;
- правил формирования cache;

необходимо:

1. обновить код;
2. выполнить SQL (если требуется);
3. полностью пересобрать cache;
4. проверить статистику Builder;
5. выполнить тестовый подбор.

---

## 8. Типичные проблемы

### Таблица пустая

Возможные причины:

- отсутствуют обязательные атрибуты;
- Catalog Standardization не выполнена;
- Builder исключил все товары.

---

### Не отображаются товары

Проверить:

- статус товара;
- цену;
- наличие;
- обязательные атрибуты;
- содержимое `pump_selector_product`.

---

### Старые данные

После изменения характеристик необходимо полностью пересобрать cache.

---

## 9. Чек-лист

После установки должно быть выполнено:

- ✅ SQL установлен
- ✅ cache построен
- ✅ Builder завершился без ошибок
- ✅ статистика Builder корректна
- ✅ таблица содержит данные
- ✅ подборщик возвращает рекомендации
- ✅ отображаются актуальные карточки товаров
- ✅ отображаются актуальные цены
- ✅ отображаются актуальные остатки

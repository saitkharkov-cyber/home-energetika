# Cache Builder

## 1. Назначение документа

Документ описывает назначение, ответственность и контракт компонента `PumpSelectorCacheBuilder`.

Он определяет:

- источник данных;
- алгоритм построения cache;
- правила отбора товаров;
- структуру выходных данных;
- архитектурные ограничения.

Документ не описывает реализацию на PHP.

---

# 2. Назначение Cache Builder

`PumpSelectorCacheBuilder` — служебный backend-компонент.

Его задача — построить техническую cache-таблицу `pump_selector_product`, содержащую только данные, необходимые для быстрого пользовательского подбора.

Builder не участвует в выборе рекомендаций.

Builder не отвечает за отображение товаров.

---

# 3. Источник данных

Единственным источником данных является стандартный каталог OpenCart.

Builder читает:

- товары;
- производителей;
- категории;
- атрибуты;
- цену;
- остаток;
- статус.

Нормализованные характеристики должны уже существовать в каталоге.

> **Важно**
>
> Builder не выполняет повторную нормализацию характеристик.

---

# 4. Поток построения cache

```text
Start rebuild()
        ↓
Очистить cache
        ↓
Загрузить товары
        ↓
Проверить обязательные атрибуты
        ↓
Проверить базовые условия
        ↓
Преобразовать значения
        ↓
Вычислить brand_priority
        ↓
Записать строку в cache
        ↓
Вернуть статистику
```

---

# 5. Базовые проверки

Товар может попасть в cache только если выполняются все обязательные условия.

Минимальные проверки:

- присутствуют обязательные атрибуты;
- цена больше нуля;
- товар активен;
- товар есть в наличии.

Конкретные проверки могут расширяться.

---

# 6. Обязательные атрибуты

Builder использует только канонические `attribute_id`.

| Attribute ID | Назначение |
|--------------|------------|
| 12 | Максимальный напор |
| 13 | Производительность |
| 15 | Напряжение |
| 44 | Диаметр насоса |

Поиск по названиям характеристик запрещён.

---

# 7. Формирование cache

Каждая строка cache должна содержать только данные, необходимые алгоритму подбора.

Обязательные поля:

- product_id
- max_head_m
- max_flow_l_min
- pump_diameter_mm
- voltage
- brand_priority
- is_eligible
- product_price
- quantity
- status

Display-данные не должны храниться в cache.

---

# 8. Brand Priority

Builder вычисляет `brand_priority` по производителю.

Текущее соответствие:

| Производитель | Priority |
|---------------|---------:|
| Pedrollo* | 10 |
| Sumoto* | 8 |
| Belamos* | 5 |
| Остальные | 0 |

Алгоритм вычисления может изменяться без изменения контракта Builder.

---

# 9. Что Builder не делает

Builder не должен:

- выбирать Best Price;
- выбирать Optimal;
- выбирать Premium;
- рассчитывать переподбор;
- выполнять ранжирование;
- получать карточки товара;
- работать с UI.

Это ответственность `PumpSelector`.

---

# 10. Возвращаемая статистика

После завершения Builder возвращает:

```php
array(
    'total_scanned' => int,
    'eligible_inserted' => int
)
```

Дополнительная статистика может быть расширена без изменения контракта.

---

# 11. Инварианты

Следующие правила должны сохраняться независимо от реализации:

- Builder читает только нормализованные данные.
- Builder не изменяет каталог.
- Builder не выполняет подбор.
- Builder строит cache полностью.
- Cache может быть полностью пересоздан в любой момент.
- Builder использует только `attribute_id`.
- Builder не хранит display-данные.

---

# 12. Что может изменяться

Может изменяться:

- SQL;
- способ чтения данных;
- batch-обработка;
- производительность;
- способ записи;
- вычисление `brand_priority`;
- состав статистики.

Не должны изменяться:

- ответственность Builder;
- источник данных;
- контракт cache;
- разделение между Builder и Selector.

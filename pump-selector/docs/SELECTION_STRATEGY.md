# Selection Strategy

## 1. Назначение

Документ описывает бизнес-логику выбора рекомендуемых насосов.

Он определяет:

- роли рекомендаций;
- критерии выбора;
- приоритеты;
- принципы принятия решений.

Документ не описывает реализацию алгоритма. Реализация может изменяться при сохранении изложенных здесь правил.

---

# 2. Общие принципы

Подборщик должен:

- рекомендовать только насосы, удовлетворяющие требованиям пользователя;
- минимизировать переподбор;
- учитывать стоимость;
- учитывать приоритет бренда;
- формировать понятную линейку рекомендаций.

Каждая следующая рекомендация должна быть качественным улучшением предыдущей.

---

# 3. Best Price

## Назначение

Самый доступный насос, полностью удовлетворяющий требованиям пользователя.

## Основная идея

Минимальная цена при соблюдении всех требований.

## Приоритеты

1. Соответствие требованиям.
2. Минимальная цена.
3. Минимальный переподбор.
4. Стабильность результата.

---

# 4. Optimal

## Назначение

Наиболее сбалансированный насос.

## Основная идея

Минимальный разумный запас характеристик без существенной переплаты.

## Приоритеты

1. Соответствие требованиям.
2. Разумный запас.
3. Минимизация переподбора.
4. Цена.
5. Приоритет бренда.

---

# 5. Premium

## Назначение

Лучшее премиальное решение.

## Основная идея

Premium одновременно должен:

- относиться к приоритетному бренду;
- являться улучшением относительно Optimal;
- иметь техническое преимущество без чрезмерного переподбора;
- оправдывать более высокую стоимость.

## Приоритеты

1. Соответствие требованиям.
2. Приоритет бренда.
3. Техническое превосходство над Optimal.
4. Разумный запас.
5. Цена.

---

# 6. Общие ограничения

Подборщик не должен:

- рекомендовать один и тот же товар в нескольких ролях;
- предлагать Premium только потому, что он единственный остался;
- выбирать насос исключительно по цене;
- выбирать насос исключительно по бренду;
- рекомендовать чрезмерно мощные модели без объективной причины.

---

# 7. Допустимые компромиссы

Допускается:

- немного более высокая цена ради лучшего бренда;
- немного больший запас ради надёжности.

Не допускается:

- многократная переплата без ощутимой пользы;
- значительный переподбор только ради бренда.

---

# 8. Инварианты

Эти правила должны сохраняться независимо от реализации алгоритма.

Изменение SQL, структуры cache, scoring, fallback или других внутренних механизмов не должно нарушать принципы, изложенные в данном документе.

---

# 9. Что может изменяться

Может изменяться:

- SQL;
- scoring;
- формулы;
- коэффициенты;
- brand_priority;
- cache;
- fallback-ветки.

Не должно изменяться:

- смысл ролей Best Price;
- смысл ролей Optimal;
- смысл ролей Premium;
- общие принципы выбора.

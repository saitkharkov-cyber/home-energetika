# Выполнено

## 1. Определена архитектурная модель Framework

Принято решение отказаться от набора отдельных скриптов в пользу универсального конвейера обработки.

Основные сущности:

- задача обработки характеристики (`Attribute Job`);
- контекст характеристики (`AttributeContext`);
- этап (`Stage`);
- движок конвейера (`Pipeline Engine`).

## 2. Создан архитектурный документ

docs/ATTRIBUTE_PIPELINE.md

Документ содержит:

- назначение конвейера;

На текущий момент документ считается базовым архитектурным контрактом Framework.

---

## 3. Архитектурные решения

Зафиксированы следующие принципы.

### Единицей обработки является

```
Attribute Job
```

### Все данные проходят через

```
AttributeContext
```

### Этапы не взаимодействуют напрямую.

Единственный способ обмена данными — `AttributeContext`.

### Расширение Framework выполняется только через добавление новых этапов.

Изменение существующего Pipeline считается исключением.

- английские названия используются только как ссылки на сущности кода (`AttributeContext`, `Pipeline Engine`, `LoadStage` и т.п.).

Это правило применять ко всей дальнейшей документации проекта.

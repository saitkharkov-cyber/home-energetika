# Apply / применение

`Apply` — это этап фактического применения подготовленных изменений.

В рамках `framework-standardization` apply означает переход от анализа, предложений или проверки к изменению состояния системы.

Apply может затрагивать:

* данные;
* SQL;
* нормализованные значения;
* файлы применения;
* рабочую логику;
* production-состояние.

Apply не должен выполняться автоматически после:

* spec;
* dry-run;
* generated proposals;
* review-chain;
* статуса `approved`;
* успешных runtime checks.

Для apply нужны:

* утверждённый contract;
* понятный apply-plan;
* явное human decision;
* отдельное подтверждение пользователя;
* проверенные ограничения безопасности.

Apply не означает просто “продолжить работу”.

Apply не означает “сгенерировать предложения”.

Apply не означает “показать, что будет изменено”.

`Approved` в review-chain не означает разрешение на apply.

`SQL apply` — частный и более опасный вид apply, при котором изменения применяются через SQL.

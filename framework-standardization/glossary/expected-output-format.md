# Expected output format / ожидаемый формат результата

`Expected output format` — это заранее заданный формат, в котором должен быть представлен результат шага.

В рамках `framework-standardization` expected output format нужен, чтобы Codex, CLI-команда или ChatGPT не выбирали форму результата самостоятельно.

Expected output format может задавать:

* вывод в консоль;
* markdown-структуру;
* таблицу;
* список предупреждений;
* список candidates;
* список unresolved cases;
* формат отчёта;
* формат файла, если создание файла явно разрешено.

Expected output format должен быть указан до выполнения шага, если форма результата важна для review или следующего gate.

Если указан console output only, результат должен быть выведен в консоль и не должен создавать файлы.

Если указан markdown output, результат должен быть пригоден для чтения человеком и ручного review.

Expected output format не означает разрешение:

* менять файлы;
* создавать runtime artifacts;
* делать commit;
* выполнять apply;
* расширять scope.

Если результат нужен для следующего шага workflow, формат должен быть стабильным и понятным без дополнительного угадывания.

# Dry-run / предварительный прогон

`Dry-run` — это предварительный прогон без применения изменений.

В рамках `framework-standardization` dry-run нужен, чтобы показать, что было бы сделано, если бы применение изменений было разрешено.

Dry-run может показывать:

* какие данные будут затронуты;
* какие значения будут преобразованы;
* какие записи попадут в apply-plan;
* какие предупреждения есть перед применением;
* какие случаи остаются нерешёнными.

Dry-run не должен менять:

* БД;
* production-данные;
* cache;
* schema;
* OpenCart runtime;
* SQL apply-файлы;
* рабочую логику проекта.

Dry-run не означает разрешение на apply.

Успешный dry-run не означает, что изменения можно применять автоматически.

После dry-run нужен отдельный gate или human decision перед apply.

Если dry-run работает с БД, режим должен оставаться readonly, пока явно не разрешён другой allowed operation mode.

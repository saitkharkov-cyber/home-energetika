### Generic / универсальный

**Generic / универсальный** — обобщённый компонент workflow, который не зашит под одну конкретную характеристику, категорию или частный кейс.

Generic-компонент получает все рабочие параметры из явного contract-файла:

- target characteristic / целевая характеристика;
- canonical attribute / каноническая характеристика;
- aliases / допустимые алиасы;
- scope / область применения;
- normalizer / нормализатор;
- safety policy / политика безопасности;
- allowed operation mode / разрешённый режим работы;
- expected output format / ожидаемый формат результата.

Generic-компонент не должен самостоятельно угадывать, какую характеристику обрабатывать, какие aliases считать допустимыми, какой scope применять и какие действия безопасны.

Пример: generic alias cleanup command может работать с “максимальным напором”, “напряжением” или другой характеристикой, если для неё заранее подготовлен explicit contract.

Важно: generic не означает automatic apply. Даже универсальный компонент проходит через contract, dry-run/review и gate перед любым изменением данных.

#### Правило универсальных команд / Generic command rule

Универсальные команды разрешены только тогда, когда их поведение полностью задано явным contract-файлом.

Универсальная команда не должна содержать зашитых бизнес-допущений под конкретную характеристику, категорию, группу товаров или частный сценарий очистки.

Если для запуска команды нужно угадывать отсутствующие aliases, scope, canonical attribute, normalizer или safety policy, команда ещё не готова к запуску.

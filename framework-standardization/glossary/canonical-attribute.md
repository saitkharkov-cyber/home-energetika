# Canonical attribute / каноническая характеристика

`Canonical attribute` — это основная утверждённая характеристика, к которой привязывается текущий процесс стандартизации.

В рамках `framework-standardization` canonical attribute выбирается из найденных кандидатов после DB-readonly discovery.

Canonical attribute может быть выбрана только после human decision.

Canonical attribute определяет:

* какой `attribute_id` считается основным;
* какие значения будут инвентаризироваться;
* какие aliases могут быть рассмотрены;
* какой contract нужно подготовить;
* для каких данных могут генерироваться proposals.

Canonical attribute нельзя выбирать автоматически только по похожести названий.

Похожие характеристики могут иметь разный смысл, разные единицы измерения или разные форматы значений.

Выбор canonical attribute не означает:

* автоматическое объединение aliases;
* разрешение на SQL apply;
* разрешение на изменение production-данных;
* разрешение на cache rebuild;
* начало implementation без отдельного подтверждения.

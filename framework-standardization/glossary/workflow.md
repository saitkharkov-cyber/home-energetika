## Workflow

**Workflow** — это рабочий процесс, то есть заранее понятная цепочка шагов:

- что делаем сначала;
- что делаем потом;
- где нужно решение человека;
- где можно передать задачу дальше.

В нашем проекте workflow — это не код и не модуль.

Это правильный порядок работы с характеристикой.

### Схема workflow в нашем проекте

```text
пользователь задаёт смысл характеристики
-> система ищет похожие характеристики в базе
-> пользователь выбирает главную canonical-характеристику
-> пользователь подтверждает aliases / дубли
-> система собирает raw values
-> пользователь утверждает единицу измерения и contract
-> система создаёт proposals
-> review-chain проверяет proposals
-> apply-plan только отдельным решением
```

### Тот же workflow по шагам

1. Пользователь задаёт смысл характеристики.
2. Система ищет похожие характеристики в базе.
3. Пользователь выбирает главную canonical-характеристику.
4. Пользователь подтверждает aliases / дубли.
5. Система собирает raw values.
6. Пользователь утверждает единицу измерения и contract.
7. Система создаёт proposals.
8. Review-chain проверяет proposals.
9. Apply-plan возможен только отдельным решением.

### Простыми словами

Workflow — это маршрут, по которому должна пройти характеристика:

от “у нас в базе хаос”  
до “у нас есть утверждённый безопасный план стандартизации”.

### Пример из жизни

Обычный заказ в интернет-магазине тоже проходит workflow:

```text
клиент оформил заказ
-> менеджер подтвердил заказ
-> склад собрал заказ
-> доставка отправила заказ
-> клиент получил заказ
```

### В нашем случае

Характеристика проходит похожий маршрут:

```text
характеристика найдена
-> похожие названия проверены
-> канон выбран человеком
-> значения собраны
-> единица измерения утверждена
-> нормализация предложена
-> review пройден
-> только потом возможен apply
```

### Главная мысль

Workflow нужен, чтобы не перепрыгнуть опасные шаги.

Например, нельзя сразу делать `config/job` или `SQL apply`, пока не пройдены:

- discovery;
- canonical selection;
- raw values inventory;
- unit contract.

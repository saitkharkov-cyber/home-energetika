# Alias / алиас

`Alias` — это дополнительное название или вариант записи, который может относиться к той же характеристике, что и canonical attribute.

В рамках `framework-standardization` alias может быть:

* дублем характеристики;
* старым названием;
* похожим названием;
* вариантом написания;
* характеристикой с тем же смыслом, но другим `attribute_id`.

Alias нельзя автоматически считать безопасным только по похожести названия.

Перед использованием alias нужно явно решить:

* относится ли он к той же характеристике;
* можно ли включить его в текущий contract;
* нужно ли его исключить;
* не отличается ли его смысл, единица измерения или формат значений.

Alias может быть включён в работу только после human decision.

Alias не означает automatic merge.

Alias не означает SQL apply.

Alias не означает разрешение менять production-данные.

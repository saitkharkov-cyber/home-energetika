# Git staging

`Git staging` — это промежуточный шаг перед commit.

Файл может быть изменён на диске, но ещё не добавлен в будущий commit. В этом случае он отображается как not staged.

После команды `git add` файл становится staged — то есть подготовлен к commit.

В рамках `framework-standardization` перед commit нужно проверять `git status` и убеждаться, что в staged попали только ожидаемые файлы текущего scope.

Если файл изменён, но не должен входить в текущий commit, его не нужно добавлять через `git add`.

Основные состояния:

- `not staged` — файл изменён, но не подготовлен к commit;
- `staged` — файл подготовлен к commit;
- `committed` — файл уже сохранён в истории Git.

Важно: `git commit` забирает только staged changes.

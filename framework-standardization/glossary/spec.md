# Spec

`Spec` — это спецификация: документ, который описывает, что нужно сделать, зачем это нужно, какие есть границы, ограничения и ожидаемый результат.

В рамках `framework-standardization` spec не является implementation.

Spec может описывать:

* цель шага;
* входные данные;
* ожидаемый результат;
* ограничения безопасности;
* запрещённые действия;
* формат вывода;
* проверки;
* границы будущей реализации.

Spec может быть подготовлена до явного подтверждения `+`.

Наличие spec не означает разрешение менять код, конфигурацию, jobs, fixture, cache или production-данные.

Implementation начинается только после отдельного явного подтверждения `+`.

Если spec используется для Codex-задачи, ChatGPT должен на её основе подготовить bounded prompt:

* какие файлы читать;
* какой один файл менять или создавать;
* что запрещено;
* какие проверки выполнять;
* какие проверки не выполнять;
* делать ли commit или не делать.

Spec нужна для того, чтобы перед implementation зафиксировать смысл, границы и правила шага.

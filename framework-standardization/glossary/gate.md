# Gate

`Gate` — это контрольная точка допуска между этапами workflow.

Gate нужен, чтобы следующий этап не начинался автоматически только потому, что предыдущий шаг уже выполнен.

В рамках `framework-standardization` через gate нельзя проходить без проверки входных условий, ограничений безопасности и явного решения, какой следующий шаг разрешён.

Gate может проверять:

* какие данные уже собраны;
* какое human decision уже принято;
* какие ограничения действуют;
* что разрешено делать дальше;
* что всё ещё запрещено.

Gate не означает SQL apply.

Gate не заменяет human decision.

Gate не даёт автоматического разрешения на implementation, apply, cache rebuild, fixture/job или изменение production-данных.

Пример текущего gate:

подготовка review-chain для generated proposals, без SQL apply.

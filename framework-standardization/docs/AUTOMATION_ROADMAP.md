# Общий план развития автоматизации стандартизации характеристик

Файл:

```text
framework-standardization/docs/AUTOMATION_ROADMAP.md
```

## 1. Назначение документа

Этот документ содержит общий план развития автоматизации проекта `framework-standardization` и перечень выполненных и оставшихся работ.

Цель проекта:

```text
перейти от отдельных ручных операций и временных скриптов
к безопасной системе анализа, управляемого изменения данных,
пакетной обработки и веб-интерфейсу администратора
```

Документ позволяет:

* видеть полный объём работ;
* понимать зависимости между крупными ступенями;
* отмечать завершённые задачи;
* видеть активные препятствия;
* определять ближайшую ограниченную задачу;
* не терять общую цель при работе над отдельной характеристикой.

Этот документ не заменяет:

* `DECISIONS.md` — источник утверждённых решений;
* `RUNTIME_CHECKS.md` — источник результатов проверок;
* `RULES.md` — правила выполнения работ;
* технические задания на отдельные компоненты;
* передачу контекста между сессиями;
* разрешение на изменение базы данных.

Каждая новая ограниченная задача, включающая изменение файлов или данных, требует отдельного пользовательского `+`.

## 2. Обозначения

| Обозначение      | Значение                                      |
| ---------------- | --------------------------------------------- |
| `[x]`            | задача завершена и проверена                  |
| `[ ]`            | задача ещё не завершена                       |
| `ЧАСТИЧНО`       | реализована только часть задачи               |
| `ЗАБЛОКИРОВАНО`  | продолжение невозможно без отдельного решения |
| `ПРИОСТАНОВЛЕНО` | направление временно отложено                 |
| `НЕ ПРИМЕНЯЕТСЯ` | задача признана неприменимой                  |

Задача считается завершённой только при выполнении всех применимых условий:

1. Реализация завершена.
2. Проверка синтаксиса пройдена.
3. Автоматические проверки пройдены.
4. Проверка на управляемой локальной копии базы выполнена, если она требуется.
5. Результат зафиксирован в постоянной документации.
6. Рабочее дерево Git приведено в ожидаемое состояние.
7. Не выполнено никаких действий за пределами разрешённой области.

## 3. Общая схема развития

```text
Ступень 1
Просмотр и анализ без изменения данных
        │
        ▼
Ступень 2
Управляемое применение изменений и очистка старых атрибутов
        │
        ▼
Ступень 3
Пакетная обработка характеристик
        │
        ▼
Ступень 4
Веб-интерфейс администратора
```

Веб-интерфейс можно вводить постепенно.

```text
Завершена ступень 1
→ веб-интерфейс просмотра без изменения данных

Завершены ступени 1 и 2
→ управляемое применение изменений через веб-интерфейс

Завершены ступени 1–3
→ пакетное управление через веб-интерфейс
```

# 8. Правила ведения плана

Документ обновляется, когда:

* завершена отдельная ограниченная задача;
* выполнена существенная проверка;
* принято решение, меняющее последовательность работ;
* появилась новая блокировка;
* блокировка устранена;
* направление приостановлено;
* задача признана неприменимой;
* изменились условия завершения ступени.

При обновлении необходимо:

1. Изменить состояние соответствующего пункта.
2. Обновить сводное состояние ступени.
3. Добавить или закрыть препятствие.
4. Проверить последовательность ближайших работ.
5. Добавить ссылку на `DECISIONS.md` или `RUNTIME_CHECKS.md`.
6. Не вставлять полный вывод команд.
7. Не превращать этот документ в журнал изменений.
8. Не удалять завершённые пункты.
9. Оставлять завершённые пункты отмеченными `[x]`.
10. Не использовать документ как разрешение на изменение базы.

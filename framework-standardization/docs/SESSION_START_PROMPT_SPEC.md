# SESSION_START_PROMPT_SPEC — спецификация bootstrap prompt

## 1. Назначение спецификации

Этот постоянный документ определяет форму будущего `docs/SESSION_START_PROMPT.md`.

Источник истины о роли и границах ответственности этого документа — `docs/DOCUMENTATION_BOUNDARIES.md`.

## 2. Назначение SESSION_START_PROMPT.md

`SESSION_START_PROMPT.md` — короткий постоянный готовый к копированию prompt, который пользователь вставляет в новый чистый ChatGPT-чат.

Он запускает bootstrap и направляет новый ChatGPT к `START_HERE.md`; startup flow самостоятельно не воспроизводит.

## 3. Структура

Prompt строится в следующем порядке: repository identity; working project и branch; ссылка на `START_HERE.md` и поручение полностью выполнить определённый там startup flow; требование представить предусмотренный этим flow отчёт; запрет изменений до отдельного пользовательского `+`; требование остановиться после отчёта.

## 4. Обязательные элементы

Prompt должен:

1. указать repository identity;
2. указать working project и branch;
3. направить прочитать `framework-standardization/docs/START_HERE.md`;
4. поручить полностью выполнить startup flow, определённый в `START_HERE.md`;
5. потребовать представить предусмотренный этим flow отчёт;
6. запретить любые изменения до отдельного пользовательского `+`;
7. потребовать остановиться после отчёта.

## 5. Запрещённое содержимое

Prompt не должен содержать:

* полный startup reading order;
* текущие SHA, target или stage;
* completed/open items;
* текущий next bounded step;
* содержимое `HANDOFF.md`;
* архитектуру проекта;
* подробные gates;
* инструкции Codex;
* lifecycle handoff;
* protected-файлы конкретной сессии;
* сведения о текущей характеристике.

## 6. Ссылки и dynamic state

Вместо дублирования startup flow prompt ссылается на `START_HERE.md` и требует полностью выполнить определённый там startup flow.

Prompt не содержит dynamic state текущей сессии. Он не добавляет текущие SHA, target, stage, completed/open items или next bounded step.

## 7. Ограничение изменений и остановка

До отдельного пользовательского `+` prompt запрещает любые file changes, implementation, commit, push и иные изменения состояния.

После восстановления контекста ChatGPT обязан показать отчёт и остановиться, не переходя к следующему bounded step.

## 8. Размер

Prompt должен оставаться коротким и пригодным для прямого копирования в новый чат.

Целевой размер — до 12 непустых строк. Жёсткий максимум — 18 непустых строк.

## 9. Validation checklist

Перед сохранением `SESSION_START_PROMPT.md` проверить:

* указан repository identity;
* указаны working project и branch;
* есть ссылка на `START_HERE.md`;
* startup flow не продублирован;
* требуется полное выполнение startup flow из `START_HERE.md`;
* запрошен предусмотренный этим flow отчёт;
* до отдельного пользовательского `+` запрещены изменения;
* после отчёта требуется остановка;
* отсутствует dynamic state;
* отсутствуют инструкции Codex и lifecycle handoff;
* соблюдён лимит размера.

## 10. Когда обновлять спецификацию

Обновлять этот документ только при изменении постоянных требований к форме или валидации `SESSION_START_PROMPT.md`.

При изменении общих ролей документов сначала обновляется `docs/DOCUMENTATION_BOUNDARIES.md`, затем эта спецификация приводится в соответствие.

Не использовать эту спецификацию для хранения текущего состояния сессии, инженерного workflow или содержимого будущего prompt.

# ТЗ: Framework автоматизированной стандартизации характеристик каталога

## Цель

Создать универсальный инструмент, позволяющий стандартизировать любые характеристики товаров без написания нового кода для каждой характеристики.

После реализации добавление новой характеристики должно сводиться к созданию небольшой конфигурации и, при необходимости, специализированного парсера.

---

# Основные принципы

Framework не изменяет данные напрямую.

Он помогает выполнить безопасную стандартизацию.

Каждый этап должен быть проверяемым.

Все изменения должны выполняться через сгенерированный SQL.

SQL не применяется автоматически.

---

# Архитектура

```text
catalog-standardization/

    config/

        attributes/

            head.yaml

            flow.yaml

            diameter.yaml

            voltage.yaml

            ...

    parsers/

        base_parser.py

        head_parser.py

        flow_parser.py

        diameter_parser.py

        voltage_parser.py

    generators/

        generate_attribute_sql.py

    reports/

    sql/generated/

    unknown-values/
```

---

# Конфигурация характеристики

Например

```yaml
attribute_id: 44

attribute_name: Диаметр насоса (мм)

category_id: 11900213

parser: diameter

type: decimal

allow_empty: false
```

Вся логика выбора характеристики должна определяться конфигурацией, а не кодом.

---

# Этапы обработки

## 1. Загрузка данных

Получить:

* product_id
* исходное значение атрибута

для выбранной категории.

---

## 2. Анализ

Построить список:

уникальных исходных значений

например

```text
98 мм

98мм.

98

98mm

100 мм

...
```

---

## 3. Нормализация

Для каждого значения вызвать соответствующий Parser.

Парсер возвращает

```python
NormalizeResult

value

status

comment
```

где

status

может быть

```text
OK

UNKNOWN

AMBIGUOUS
```

---

## 4. Формирование SQL

SQL генерируется только для записей

```text
OK
```

Например

```sql
UPDATE ...

SET text='98'

...
```

---

## 5. Формирование отчета

Автоматически создать

```text
report.md
```

например

```text
Всего товаров

186

Уникальных значений

43

Автоматически обработано

39

Требуют ручной проверки

4
```

---

## 6. Формирование списка неизвестных значений

Создать

```text
unknown_values.csv
```

Например

```text
product_id

исходное значение

причина
```

---

# Parser API

Каждый parser обязан реализовать

```python
normalize(raw_value)
```

и вернуть

```python
NormalizeResult
```

Запрещается изменять другие parser.

Каждый parser отвечает только за свою характеристику.

---

# Базовый Parser

Общий класс должен содержать

* очистку пробелов

* замену запятых

* unicode

* служебные методы

---

# Generator

Generator отвечает только за

создание SQL

и не содержит логики нормализации.

---

# SQL

Все SQL помещаются в

```text
sql/generated/
```

---

# Безопасность

Framework никогда самостоятельно не изменяет БД.

Он только генерирует SQL.

Применение SQL выполняет разработчик после проверки.

---

# Расширяемость

Добавление новой характеристики должно требовать:

1.

создать yaml

2.

при необходимости создать parser

3.

запустить generator

без изменения ядра Framework.

---

# Цель проекта

После реализации Framework стандартизация новой характеристики должна занимать минуты вместо часов.

Большая часть работы должна выполняться автоматически.

Ручная работа должна сводиться только к разбору действительно неоднозначных значений.

---

## Архитектурная идея

Я бы добавил еще один принцип, который, как мне кажется, станет ключевым для долговременного развития проекта.

Framework не должен быть "генератором SQL". Это лишь один из его выходов.

Framework должен стать **конвейером обработки характеристик**:

```text
Источник данных
        ↓
Загрузка
        ↓
Парсер
        ↓
Валидация
        ↓
Отчет
        ↓
Генерация SQL
```

Тогда в будущем вместо SQL можно будет легко добавить другие выходы, например экспорт в Excel, HTML-отчеты или даже прямую интеграцию с системой контроля качества каталога, не меняя саму логику обработки данных. Именно такая архитектура, на мой взгляд, сделает инструмент действительно универсальным.
